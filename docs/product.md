# ScrcpyBluetooth - 产品介绍

## 一句话定位

**无需网络、端到端加密的 Android 对 Android 屏幕镜像与远程控制工具。**

## 核心亮点

### 1. 纯蓝牙/USB，零网络依赖

不需要 Wi-Fi、不需要移动数据、不需要任何服务器。两台 Android 设备通过蓝牙配对即可实现屏幕镜像和远程控制。飞机上、地下室、野外——有蓝牙就能用。

### 2. 端到端加密，安全第一

采用与 Signal 同级别的加密方案：
- ECDH P-256 密钥交换
- AES-256-GCM 认证加密
- HKDF-SHA256 密钥派生
- TOFU 设备认证模型

中继模式下，中继设备只转发密文，无法解密任何内容。

### 3. Android 对 Android，被控端无需 ROOT

被控端通过 `app_process` 以 Shell 权限运行，不需要 ROOT。控制端是标准 Android APP，安装即用。

### 4. 丰富的附加功能

| 功能 | 说明 |
|------|------|
| 剪贴板同步 | 双向实时剪贴板传递 |
| 文件传输 | 支持大文件分块传输、断点续传 |
| 文件夹增量同步 | SHA-256 块级哈希比较，只传差异部分 |
| 分享转发 | 系统分享菜单一键转发文本/文件到被控端 |
| 中继模式 | 第三台设备做桥梁，延伸蓝牙距离 |

### 5. 资源友好，后台稳定

- 256 色编码 + LZ4 压缩，蓝牙带宽下流畅运行
- 增量帧传输，画面不变时零开销
- 前台服务 + WakeLock 自动续期，后台不被杀
- 权限优雅降级，拒绝权限不崩溃

## 使用场景

| 场景 | 说明 |
|------|------|
| **远程协助** | 帮父母/朋友操作手机，无需在同一地点（蓝牙范围内） |
| **设备演示** | 会议/课堂上，用一台手机控制另一台进行演示 |
| **多机操控** | 一台手机控制另一台运行特定任务 |
| **离线环境** | 飞机、矿井、信号盲区等无网络场景的设备管理 |
| **安全敏感** | 不信任网络环境时，蓝牙直连 + 端到端加密确保数据不出设备 |

## 技术规格

- **协议版本**：v1
- **编码格式**：256 色索引 + XOR 增量 + LZ4 压缩
- **加密**：AES-256-GCM，12 字节 nonce（4B base + 8B counter）
- **传输**：蓝牙 RFCOMM / USB ADB 抽象 socket
- **帧策略**：关键帧每 150 帧，调色板每 90 帧重建，脏区 >60% 升级关键帧
- **最低系统**：Android 9.0 (API 28)
