## 安装与准备

### 被控端 (C) 准备

被控端需要 **Root 权限**，因为服务端通过 `su` 命令以 `app_process` 方式启动，需要 shell 级权限访问 Android 隐藏 API。

1. 安装 APK 到被控端设备
2. 确保设备已获得 Root（Magisk、KernelSU 等）
3. 首次启动时，应用会检测 Root 可用性并提示

### 控制端 (A) / 中继端 (B) 准备

控制端和中继端**不需要 Root**。

1. 安装 APK
2. 授予蓝牙相关权限（发现、连接、位置）
3. 如使用 USB 方式，需在电脑上执行 `adb forward` 命令

## 基本使用流程

### 第一步：选择角色

启动应用后，在主界面选择角色：

- **控制端 (A)**：显示远程画面，发送触摸操作
- **中继端 (B)**：透明转发 A↔C 之间的数据
- **被控端 (C)**：运行服务端，捕获屏幕并接受远程控制

### 第二步：选择传输方式

- **蓝牙**：无需线缆，两台设备需先完成蓝牙配对
- **USB ADB**：通过 USB 线缆和 ADB forward 连接，延迟更低

### 第三步：建立连接

**蓝牙方式：**
1. 被控端选择"被控端"角色 → 点击"启动服务端"
2. 控制端选择"控制端"角色 → 从已配对设备列表选择被控端 → 连接

**USB ADB 方式：**
1. 两台设备通过 USB 连接到电脑（或 OTG 直连）
2. 在电脑上执行：`adb forward tcp:27183 localabstract:scrcpy_bt`
3. 被控端启动服务端（选择 USB 传输）
4. 控制端选择 USB 传输 → 连接

### 第四步：自动握手与加密

连接建立后，双端自动完成：
1. 握手交换设备信息和屏幕分辨率
2. ECDH 密钥交换建立加密通道
3. 首次连接时，被控端弹出认证确认对话框
4. 认证通过后，控制端开始接收屏幕画面

## 屏幕镜像

### 查看远程画面

控制端自动接收被控端的 256 色编码帧，解码后显示在 SurfaceView 上。画面保持原始分辨率，SurfaceView 自动按比例缩放适配屏幕。

### 触摸控制

在控制端的画面上直接触摸即可操作被控端：
- 支持单点触摸（点击、滑动、长按）
- 支持多点触控（缩放、旋转手势）
- 触摸坐标自动按分辨率比例换算

### 物理按键

控制端底部导航栏提供快捷按钮：
- **返回**：模拟返回键
- **主页**：模拟 Home 键
- **最近任务**：打开最近应用列表
- **熄屏**：远程关闭/打开被控端屏幕
- **音量+/-**：调节被控端音量

### 运行时模式切换

通过控制端菜单可以在运行中切换：
- **镜像模式 → 虚拟显示**：创建虚拟屏幕，物理屏幕关闭
- **虚拟显示 → 镜像模式**：释放虚拟屏幕，回到物理屏幕捕获

## 虚拟显示模式

### 什么是虚拟显示模式

虚拟显示模式在被控端创建一个独立的"第二屏幕"，类似 Windows 远程桌面：

- 远程用户在虚拟屏幕上操作，拥有完整的应用界面
- 物理屏幕保持关闭或显示锁屏
- 设备旁的人无法看到或操作远程会话
- 无需解锁物理屏幕，不会暴露设备

### 启用方法

1. 在被控端的"被控端"界面，打开**虚拟显示模式**开关（Android 10+ 默认开启）
2. 可选：指定虚拟显示上启动的应用（组件名格式，如 `com.android.settings/.Settings`）
3. 可选：开启 FLAG_SECURE 绕过（仅在需要查看银行/支付应用时使用）
4. 点击"启动服务端"

### 虚拟显示高级选项

**启动应用**：虚拟显示刚创建时没有 Launcher，默认为黑屏。系统会自动尝试在虚拟显示上启动一个应用来解决黑屏问题。尝试顺序：
1. 用户指定的应用
2. 系统 HOME Intent
3. 已知启动器（Pixel、三星、小米、OPPO、Vivo、华为、一加）
4. 系统设置 (Settings) 作为兜底

**FLAG_SECURE 绕过**：某些应用（如银行、支付、DRM 视频）使用 `FLAG_SECURE` 标志阻止截屏/录屏。开启此选项可以绕过该保护，在虚拟显示上查看这些应用。会话结束后自动恢复。

### 要求

- Android 10+ (API 29)
- Root 权限
- Android 9 设备自动回退到镜像模式 + 强制解锁

### 回退机制：强制解锁

对于不支持虚拟显示的设备（Android 9），可以使用强制解锁作为回退方案：

1. 开启"强制解锁"开关
2. 选择锁屏凭据类型（PIN/密码/图案/无）
3. 输入锁屏凭据（仅存储在本机，不会传输）

> **安全警告**：强制解锁模式下，远程解锁后任何能接触到设备的人都能直接使用。虚拟显示模式没有此问题。

## 剪贴板传递

在控制端通过菜单进入"剪贴板传递"界面：

- **发送到远程**：将本地剪贴板内容发送到被控端
- **从远程获取**：获取被控端剪贴板内容到本地
- **刷新**：刷新远程剪贴板显示

剪贴板内容通过加密通道传输。

## 文件传输

在控制端通过菜单进入"文件传输"界面：

### 发送文件

1. 点击"选择文件"或"选择文件夹"
2. 选择要发送的文件
3. 文件自动进入传输队列
4. 实时显示传输进度

### 接收文件

被控端可以向控制端推送文件，控制端自动接收并保存。

### 断点续传

如果传输中断（蓝牙断连、切换传输方式等），重新连接后系统会自动检测并恢复未完成的传输。

## 分享转发

### 控制端 → 被控端

1. 在控制端任意应用中使用"分享"功能
2. 在分享列表中选择"Share to Remote"
3. 确认发送
4. 内容（文本/文件）自动通过加密通道发送到被控端

### 被控端配置

在被控端的"分享转发配置"界面：
1. 开启分享转发功能
2. 指定目标应用的组件名（如 `com.example.app/.ShareActivity`）
3. 保存配置

收到分享内容后，被控端自动调用目标应用的 ACTION_SEND Intent。

## 文件夹同步

文件夹同步功能参考 Syncthing 设计，支持块级增量同步。

### 基本配置

1. **本地文件夹**：选择或输入本地路径
2. **远程文件夹**：输入被控端的文件夹路径（如 `/sdcard/Documents`）
3. **文件夹类型**：
   - 仅发送：本地更改推送到远程
   - 仅接收：远程更改拉取到本地
   - 发送和接收：双向同步

### 文件版本控制

当文件被同步操作覆盖或删除时，旧版本如何处理：

| 类型 | 说明 |
|------|------|
| 无 | 直接覆盖，无版本保留 |
| 回收站 | 旧版本移到 `.stversions/` 目录，可配置自动清理天数 |
| 简单 | 保留最近 N 个旧版本，带时间戳命名 |
| 交错 | 按时间间隔保留版本（越旧间隔越大） |
| 外部 | 调用自定义命令处理旧版本 |

### 电源条件

- **仅充电时同步**：避免同步消耗电池
- **最低电量**：电量低于阈值时暂停同步
- **遵守省电模式**：系统省电模式开启时暂停
- **同步时间窗口**：限定同步在特定小时段内进行

### 高级设置

- **重新扫描间隔**：定期全量扫描文件变化（默认 3600 秒）
- **文件系统监视器**：实时检测文件变化（FileObserver）
- **忽略删除**：同步不传播删除操作
- **拉取顺序**：随机/字母/最小优先/最大优先/最旧优先/最新优先
- **块大小**：分块大小（默认 128KB）
- **最小可用磁盘空间**：磁盘空间不足时暂停同步

### .syncignore

在同步文件夹根目录创建 `.syncignore` 文件，支持 glob 模式：

```
# 忽略临时文件
*.tmp
*.swp
~*

# 忽略目录
.git/
node_modules/
build/

# 忽略特定路径
secret.txt
config/local.properties
```

通过应用内"编辑 .syncignore"按钮可以直接编辑。

## 传输方式

### 蓝牙 RFCOMM

- **优点**：无需线缆，适合移动场景
- **缺点**：带宽有限（约 3Mbps），延迟较高
- **配对**：首次连接需先在系统设置中完成蓝牙配对
- **UUID**：`fa87c0d0-afac-11de-8a39-0800200c9a66`

### USB ADB

- **优点**：带宽充足，延迟低
- **缺点**：需要 USB 线缆和 ADB 环境
- **连接方式**：通过 `adb forward` 将 TCP 端口映射到 Unix 抽象 socket

```bash
# 在电脑上执行
adb forward tcp:27183 localabstract:scrcpy_bt
```

### 传输切换

运行中可以无缝切换传输方式（如从蓝牙切换到 USB），加密密钥和会话状态保持不变。切换流程：

1. 控制端发送 TRANSPORT_SWITCH 请求
2. 服务端确认就绪
3. 在新传输上建立连接
4. 发送会话验证
5. 服务端确认验证通过
6. 切换流式数据到新连接
7. 关闭旧连接

## 安全与认证

### 端到端加密

所有数据传输均经过 AES-256-GCM 加密：
- 密钥通过 ECDH P-256 + HKDF-SHA256 协商
- 每条消息使用递增 nonce，防止重放攻击
- 蓝牙额外有 SSP 链路层加密

### 设备认证 (TOFU)

首次连接时（Trust On First Use）：
1. 被控端弹出认证确认对话框
2. 显示控制端设备名称和请求的功能
3. 用户选择"允许"或"拒绝"
4. 允许后设备被加入信任列表
5. 后续连接自动认证

### 按功能授权

每个功能独立授权：
- 屏幕镜像
- 剪贴板
- 文件传输
- 文件夹同步
- 分享转发

### 信任管理

在"受信任的设备"界面可以查看和撤销已授权设备。

## 服务端命令行参数

被控端的 `ServerMain` 支持以下命令行参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--transport=bluetooth\|usb` | 传输方式 | `bluetooth` |
| `--socket-name=NAME` | Unix 抽象 socket 名 (USB) | `scrcpy_bt` |
| `--max-fps=N` | 最大帧率 | `30` |
| `--virtual-display=true\|false` | 虚拟显示模式 | `false` |
| `--start-app=COMPONENT` | 虚拟显示启动应用 | 自动检测 |
| `--bypass-secure-flag=true\|false` | 绕过 FLAG_SECURE | `false` |
| `--force-unlock=true\|false` | 强制解锁 (回退方案) | `false` |
| `--credential-type=none\|pin\|password\|pattern` | 锁屏凭据类型 | `none` |
| `--credential=VALUE` | 锁屏凭据值 | 无 |
| `--share-target=COMPONENT` | 分享转发目标组件 | 无 |

完整启动命令示例：

```bash
su -c "CLASSPATH=/data/data/com.scrcpybt.app/cache/server.jar \
  app_process / com.scrcpybt.server.ServerMain \
  --transport=bluetooth \
  --virtual-display=true \
  --start-app=com.android.settings/.Settings \
  --max-fps=30"
```

## 故障排查

### 无法启动服务端

- **"Root access required"**：被控端没有 Root 权限。请安装 Magisk 或 KernelSU
- **server.jar 提取失败**：检查应用的 assets 目录是否包含 server.jar

### 蓝牙连接失败

- 确认两台设备已在系统蓝牙设置中完成配对
- 确认蓝牙权限已授予（Android 12+ 需要 BLUETOOTH_CONNECT）
- 尝试重新配对设备

### USB 连接失败

- 确认已执行 `adb forward tcp:27183 localabstract:scrcpy_bt`
- 确认 USB 调试已开启
- 检查 ADB 连接：`adb devices` 应显示设备

### 虚拟显示黑屏

- 指定 `--start-app` 参数启动一个具体应用
- 确认设备为 Android 10+
- 部分 ROM 可能不完整支持虚拟显示 API

### 画面卡顿

- 蓝牙带宽有限，可降低帧率（`--max-fps=15`）
- 切换到 USB 传输获得更好性能
- 静态画面时帧差编码会自动跳过空帧

### 文件夹同步不工作

- 确认远程路径存在且可访问
- 检查磁盘可用空间是否满足最小要求
- 检查电源条件（充电状态、电量、时间窗口）
- 查看同步日志获取错误详情
